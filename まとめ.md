# 掲示板ビューアー実装 まとめ

## 作業概要
指定された掲示板サイト（bakusai.com）を見やすく使いやすいビューアーとして再構築するアプリケーションを実装しました。

### 実装機能
1. **マウスオーバーレスプレビュー**
   - `>>番号` 形式のアンカーリンクをホバーするとポップアップで該当レスを表示
   - ポップアップはレス番号、投稿者、日時、内容を表示

2. **CSV ダウンロード**
   - スレッド全体を CSV 形式でエクスポート
   - 列：レス番号、日時、投稿者、内容
   - UTF-8 BOM 付きで Excel での文字化け対応

3. **見やすい UI**
   - レスポンシブデザイン（PC・タブレット・モバイル対応）
   - サイドバーでスレッド ID 入力フォーム
   - レス数、エラーメッセージ表示

## 技術スタック

### バックエンド
- **Node.js + Express**：Web サーバー
- **axios**：HTTP リクエスト
- **cheerio**：HTML パース / スクレイピング
- **cors**：CORS 対応
- **dotenv**：環境変数管理

### フロントエンド
- **React**：UI フレームワーク
- **Vite**：ビルドツール（高速開発サーバー）

## 起動方法

### バックエンド起動
```bash
cd bulletin-viewer
npm start
```
サーバーが `http://localhost:3001` で起動します。

### フロントエンド起動（別ターミナル）
```bash
cd bulletin-viewer/client
npm run dev
```
ブラウザが `http://localhost:5173` で起動します。

## API エンドポイント

### GET /api/thread/:id
指定したスレッド ID のデータを JSON で返す
- **レスポンス**：`{ success, data: { id, title, posts: [...], postCount, scrapedAt } }`

### GET /api/download/:id
指定したスレッドを CSV ファイルで返す
- **ダウンロード**：`thread_{id}.csv`

### GET /api/health
ヘルスチェック

## 使用方法

1. バックエンド・フロントエンドを両方起動
2. ブラウザでフロントエンド URL にアクセス
3. 左側のフォームにスレッド ID を入力（例：`13309333`）
4. 「読み込む」ボタンをクリック
5. スレッドが表示される
6. レス内の `>>番号` にマウスオーバーするとプレビュー表示
7. 「CSV ダウンロード」ボタンでエクスポート可能

## 特記事項

### スクレイピングについて
- `scraper.js` のセレクターは対象サイトの HTML 構造に基づいています
- 実際のサイトにアクセスしてセレクターを調整する必要がある場合があります
- User-Agent を設定してリクエスト送信（ブロック対策）

### パフォーマンス対策
- メモリ内キャッシュ機能：同じスレッドの重複リクエストを 1 時間キャッシュ
- 大型スレッド対応：ページネーション実装は今後の拡張予定

### セキュリティ
- XSS 対策：フロントエンドで HTML エスケープ
- CORS 対応：クロスオリジンアクセスに対応
- レート制限：今後の拡張予定
